# Track Service Front

Веб-интерфейс для загрузки геопространственных изображений, их разбивки на тайлы и просмотра на интерактивной карте.

## Стек

- **React 18** + React Router v5
- **Ant Design 4** — UI-компоненты
- **OpenLayers 10** — интерактивная карта и тайловый рендеринг

## Запуск

```bash
npm install
npm start
```

Приложение откроется на `http://localhost:3000`. 

## Настройка подключения к backend-серверу (сейчас используется мок-сервер)

Откройте файл `package.json` и измените параметр `proxy` на нужный вам адрес сервера.

```json
"proxy": "http://tracks.ai.wfront.svk.sils.local/"
```

После изменения сохраните файл и перезапустите dev-сервер (`npm start`).

## Структура проекта

```
src/
├── components/          # UI-компоненты (только JSX, без бизнес-логики)
│   ├── AppHeader/       # Шапка с названием и кнопкой выхода
│   ├── CardSidebar/     # Боковая панель со списком карточек и пагинацией
│   ├── MapArea/         # Область карты с тайлами / статичным изображением
│   ├── UploadModal/     # Модальное окно загрузки файла
│   ├── AuthPage/        # Страница авторизации (форма логина)
│   ├── AuthCheck/       # HOC — проверяет токен перед рендером защищённых страниц
│   └── AtomSpinner/     # Анимированный спиннер загрузки
│
├── hooks/               # Вся логика с состоянием
│   ├── useMap.js        # Инициализация карты, тайловые и статичные слои
│   └── useCards.js      # Загрузка, удаление, загрузка файлов, polling тайлов
│
├── services/
│   └── api.js           # Все fetch-запросы к серверу (чистые функции)
│
├── utils/
│   └── mapHelpers.js    # Вспомогательные функции (zoom, размеры, viewFit)
│
├── App.jsx              # Корневой компонент — подключает хуки, собирает layout
├── AppRouter.jsx        # Роутинг: /login → AuthPage, / → AuthCheck + App
├── index.js             # Точка входа
└── styles.css           # Глобальные стили
```

## Логика разделения

| Слой | Что делает | Пример |
|------|-----------|--------|
| `components/` | Рендерит UI, принимает пропсы | `<CardSidebar cards={...} />` |
| `hooks/` | Управляет состоянием, вызывает сервисы | `useCards`, `useMap` |
| `services/` | Делает HTTP-запросы, возвращает данные | `fetchCardsMetadata()` |
| `utils/` | Чистые функции, без side-effects | `getDimsFromLevels()` |

## Как это работает

1. Пользователь логинится → токен сохраняется в `localStorage`
2. `AuthCheck` проверяет токен при каждом переходе на защищённый маршрут
3. На главной странице загружается список изображений с сервера
4. При клике на карточку — открывается тайловая карта через OpenLayers
5. Новое изображение можно загрузить через кнопку или drag & drop — сервер разбивает его на тайлы и возвращает манифест
